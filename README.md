##  Постановка задачи
Задача заключается в предсказании склонности пользователя к определенному секретному событию на основе векатора анонимизированных признаков.

* **Тип задачи:** Бинарная классификация с ранжированием объектов.
* **Формат вывода:** Непрерывный скор (вероятность/степень склонности), а не жесткий класс.
* **Целевая метрика:** `ROC-AUC` (критически важен корректный порядок пользователей).
* **Главная сложность:** Экстремальный дисбаланс классов и сильный шум в данных.

---

##  Данные и особенности
* **Обучающая выборка (`train`):** 247 972 строк (размер ~2.5 GB).
* **Тестовая выборка (`test`):** 106 274 строк (размер ~1.1 GB).
* **Количество признаков:** 1367 анонимизированных колонок (семантика отсутствует).
* **Баланс классов:** Положительный класс (класс 1) составляет всего **1.35%**.

### Основные вызовы:
1. Одиночные признаки сами по себе слабые; полезный сигнал извлекается преимущественно из их комбинаций.
2. Данные содержат тяжелые хвосты распределений, выбросы и нестабильные распределения.

---

##  Разведочный анализ данных (EDA)
В рамках базовой и визуальной диагностики было выполнено:
* **Анализ качества:** Поиск пропусков, дубликатов строк (`train`/`test`) и квазиконстантных колонок.
* **Фильтрация шума:** Оценка связи отдельных признаков с таргетом через корреляции, single-feature AUC и Feature Importance.
* **Контроль сдвига (Drift):** Выявление признаков со смещением распределений между `train` и `test`.
* **Генерация Meta-features:** Обнаружено, что распределение *количества нулей* и *уникальных значений* в строке имеет высокую разделительную способность по классам и может использоваться как мощный мета-признак.

---

##  Feature Engineering & Feature Spaces
Вместо одного фиксированного датасета, пайплайн генерирует и тестирует несколько специализированных пространств признаков:

* `clean dataset` — базовый очищенный набор (без дублей, квазиконстант и явного шума).
* `top300 / top500 / top700` — наборы лучших колонок по комбинированному рангу важности.
* `stable / nodrift` — выборки, прошедшие жесткий контроль устойчивости распределений между train и test.
* **Преобразования:** Частотное кодирование (`frequency-encoded`), ранговые трансформации (`rank-transformed`) для стабильности порядка, а также Out-Of-Fold (OOF) Target Encoding без утечки данных.
* **Агрегации:** Компоненты PCA/SVD (включая логику по маскам ненулевых значений) и простые комбинации (`interaction features`) сильных признаков.

##  Схема валидации
Для предотвращения переобучения и обеспечения воспроизводимости результатов использовалась строгая многоэтапная схема:

1. **Разбиение (Train/Validation):** Стратифицированное деление обучающей выборки с сохранением пропорции редкого класса.
2. **Настройка (Tune):** Подбор гиперпараметров моделей строго по локальной Validation / OOF ROC-AUC (без подглядывания в test).
3. **Переобучение (Refit):** Фиксация лучшей конфигурации и финальное обучение модели на полном наборе данных `train + validation` для максимального использования разметки.
4. **Ансамблирование (Blend):** Взвешенное смешивание моделей, возвращающих разный порядок объектов и имеющих различные пространства признаков.

##  Одиночные модели
Было протестировано более десятка различных алгоритмов (бустинги, нейросети, линейные модели). Результаты лучших одиночных решений на тестовой выборке:

| Модель / Feature Space | Test ROC-AUC | Комментарий |
| :--- | :---: | :--- |
| **LightGBM** (top500 engineered) | **0.69074** | **Лучший single-baseline** |
| **LightGBM DART** | 0.68927 | Стабильный и глубокий бустинг |
| **FLAML AutoML** | 0.68439 | Сильный автоматический baseline |
| **XGBoost** (PCA/SVD stack) | 0.68315 | Хорошо отработал на сжатых компонентах |
| **CatBoost** (linear stack) | 0.68306 | Использовался для линейных признаков |
| **TabM** (magic meta) | 0.65034 | Табличная глубокая модель на мета-фичах |
| **Logistic Regression** (linear stack)| 0.66887 | Быстрая линейная модель для разнообразия бленда |

##  Ансамблирование и Блендинг (Решающий этап)
Поскольку разные архитектуры улавливали разные паттерны (бустинги — нелинейные связи, линейные модели и TabM — структуру мета-признаков), смешивание их предсказаний (`probability` и `rank`) дало ключевой буст по метрике.

Веса подбирались с помощью жадного поиска (`greedy search`) по OOF, оптимизации по сетке и подходов `power / overdrive blends`.

### Результаты блендинга на `test`:
| Бленд | Test ROC-AUC | Описание / Особенности |
| :--- | :---: | :--- |
|  **Overdrive blend** | **0.71513** | **Финальное решение. Core-модели + FLAML** |
| Surgical blend | 0.71506 | Тонкая ювелирная настройка весов |
| Tuning blend | 0.71504 | Сетка по степенному (power) параметру |
| Upgrade loop | 0.71358 | Итеративное добавление моделей по одной |
| Mass blend | 0.70947 | Смешивание 9 разнородных моделей |

##  Главные выводы проекта
1. **EDA как фильтр:** Качественный предварительный анализ уберёг от утечек данных и отсек нестабильные (drift) фичи.
2. **Многообразие признаков:** Успех обеспечил не один гигантский датасет, а построение множества специализированных пространств.
3. **Валидация решает:** Строгий OOF/Validation контроль позволил стабильно расти по метрике без риска переобучения.
4. **Сила блендинга:** Объединение сильного градиентного бустинга с линейными стекерами и AutoML-решениями подняло итоговый ROC-AUC с **0.69074** до победных **0.71513**.
